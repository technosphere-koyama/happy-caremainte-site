/*
 * module
 * 
 * 機能としてモジュール化できる処理を纏めておくところ
 * ※モジュールという程大きくない場合はlibraryに配置すること！
 * 
 * moduleの使用について
 * 
 *  モジュール毎にディレクトリ分けを行うこと
 *  外部が作ったモジュールは「thirdParty」に配置すること
 * 
 *  moduleディレクトリに入れておけば、階層が深くてもautoloadの対象となる
 *  ※モジュールトップディレクトリの頭に「_」を入れておけば、autoload対象から外れる
 *  　使用しない場合は頭に「_」を入れておくこと
 *  　これでメモリの無駄遣いはなくなる
 * 
 * ソース配置のルール
 * 
 *	モジュールディレクトリ直下に「preLoadConf」ディレクトリを配置すること
 *	更にその直下に「common」「local」「release」ディレクトリを配置すること
 *	※これらのうち1つでもかけるとエラーとなる
 *	
 *	「common」の中に配置するソースは最初にロードされる
 *	
 * 
 */
